☰ Audio Engine

Low-latency JUCE AudioEngine: pick driver (device type), optional sample rate, then output device — Apply opens an output stream (silence or optional 440 Hz test tone on F32 devices). Input section can start a capture stream (samples discarded until metering/DSP). engine_state returns output `stream` and `input_stream`. Optional buffer_frames applies to both directions. Playback output & EQ mirror the floating player; VST3/AU inserts run after the built-in EQ when you Apply inserts (stop the output stream first).
Engine status AudioEngine version, output stream, and input capture status (read-only).

☰ Output device

Driver JUCE audio device type (CoreAudio, ASIO, JACK, etc.). Changing driver stops active streams and refreshes device lists.
Sample rate Optional fixed sample rate for start_output_stream and start_input_stream. Options come from the selected output device; leave as driver default if unsure.
Output device System output device for the audio engine
Output buffer (frames) Optional fixed buffer for start_output_stream. The list comes from the selected output device’s supported sizes (get_output_device_info.buffer_sizes). Choose Driver default to omit buffer_frames.
Library playback spectrum FFT window length and how many bands are sent in playback_status. Larger windows cost more CPU; fewer bands reduce IPC payload. Off skips FFT in the engine.

    ☰ Scan Behavior

    Default Plugin Type Filter Plugin type to show by default after scanning
    Auto-Scan on Launch Automatically scan for plugins when the app starts
    Folder Watch (Auto-Scan) Watch scan directories for new/changed files and auto-index them
    Folder Watch Directories Paths to watch (one per line). Leave empty to use plugin scan dirs plus all library scan dirs below. If you add any path here, only this list is watched.
    Incremental Unified Scan When on, skip subtrees whose directory mtime is unchanged since the last complete unified scan (faster). Turn off to always walk the full tree.
    Auto-Check Updates on Launch Automatically check for plugin updates after scanning
    KVR check on startup When on, run one plugin update check after the app loads the plugin database (separate from “after auto-scan”). Default off.
    Background analysis (auto) When on, run BPM/Key/LUFS batches as the library loads and scans complete. Default off — use Run analysis for manual runs.
    Byte-duplicate scan (auto) When on, start the SHA-256 byte-duplicate scan after the app finishes loading library counts (default off).
    Fingerprint cache (auto) When on, start fingerprint cache build for the audio library after load (CPU-heavy; default off).
    PDF scan (auto) When on and PDF scan roots are configured, run Scan PDFs once at launch if the indexed PDF count is zero (default off).
    PDF metadata (auto on launch) When on, start a full background PDF page-count pass once after startup (independent of the PDF tab). Default off.
    Background PDF metadata (auto) When on, resolve PDF page counts in the background while the PDF tab is open (including on launch if that tab is restored). When off, use Command Palette → Start PDF metadata for a full pass; the table still shows values already in the database.
